# Σύστημα Διαχείρισης Κρατήσεων - Οδηγίες Εγκατάστασης και Λειτουργίας

Το παρόν έγγραφο αφορά τις οδηγίες εγκατάστασης, αρχικοποίησης δεδομένων (Seeding) και χρήσης του Συστήματος Διαχείρισης Κρατήσεων, με σκοπό την ομαλή αξιολόγησή του. Το σύστημα διαχωρίζεται σε Backend (Laravel 12) και Frontend (React.js / Vite).

## Προαπαιτούμενα (Requirements)
Για την εκτέλεση του συστήματος απαιτούνται τα παρακάτω:
1. PHP (Έκδοση 8.2 ή νεότερη)
2. Composer (PHP Package Manager)
3. Node.js (Έκδοση 18+ ή 20+) και npm
4. MySQL (ή άλλο RDBMS υποστηριζόμενο από το Laravel, π.χ. SQLite για γρήγορη δοκιμή προτείνεται)
5. Λογαριασμός GitHub (Για την ταυτοποίηση OAuth)

---

## Οδηγίες Εγκατάστασης (Setup)

### 1. Εγκατάσταση Backend (Laravel)

1. Ανοίξτε το τερματικό και πλοηγηθείτε στον φάκελο `backend`.
2. Εγκαταστήστε τις βιβλιοθήκες της PHP:
   ```bash
   composer install
   ```
3. Δημιουργήστε το αρχείο παραμέτρων περιβάλλοντος `.env`:
   ```bash
   cp .env.example .env
   ```
4. Ενημερώστε το αρχείο `.env` με τα δεδομένα της βάσης δεδομένων (DB_CONNECTION, DB_DATABASE, κ.λπ.) και συμπληρώστε τα GitHub OAuth credentials για να λειτουργεί η είσοδος μέσω GitHub:
   ```env
   GITHUB_CLIENT_ID=το_δικο_σας_client_id
   GITHUB_CLIENT_SECRET=το_δικο_σας_client_secret
   GITHUB_REDIRECT_URI=http://localhost:8000/api/auth/github/callback
   ```
   *Σημείωση: Για να αποκτήσετε Client ID/Secret, πρέπει να δημιουργήσετε ένα "OAuth App" στα "Developer Settings" του λογαριασμού σας στο GitHub.*
5. Δημιουργήστε το Application Key:
   ```bash
   php artisan key:generate
   ```
6. Δημιουργήστε τους δομικούς πίνακες στη βάση και γεμίστε το σύστημα με δοκιμαστικά δεδομένα (Seeders):
   ```bash
   php artisan migrate:fresh --seed
   ```
   *Με την παραπάνω εντολή δημιουργούνται αυτόματα ο χρήστης-Διαχειριστής, Αίθουσες, Ταινίες, και δοκιμαστικές κρατήσεις.*
7. Εκκινήστε τον διακομιστή του Backend:
   ```bash
   php artisan serve
   ```
   *Ο διακομιστής θα τρέχει στη διεύθυνση: http://localhost:8000*

### 2. Εγκατάσταση Frontend (React/Vite)

1. Ανοίξτε ένα νέο παράθυρο τερματικού και πλοηγηθείτε στον φάκελο `frontend`.
2. Εγκαταστήστε τα πακέτα Node:
   ```bash
   npm install
   ```
3. Εάν υπάρχει αρχείο `.env.example`, αντιγράψτε το σε `.env` και βεβαιωθείτε ότι το URL του Backend είναι σωστό (VITE_API_URL=http://localhost:8000).
4. Εκκινήστε τον React διακομιστή ανάπτυξης:
   ```bash
   npm run dev
   ```
   *Ο διακομιστής θα τρέχει στη διεύθυνση: http://localhost:5173* (Επισκεφτείτε αυτή τη διεύθυνση από τον browser).

## Τρόπος Λειτουργίας Συστήματος

Μόλις το σύστημα ξεκινήσει, οι βασικές ροές λειτουργίας είναι οι εξής:

### Α. Διαχείριση Συστήματος (Admin Dashboard)
Για πρόσβαση στο διαχειριστικό περιβάλλον, συνδεθείτε μέσω της σελίδας Login (`http://localhost:5173/login`) με τα διαπιστευτήρια (Admin Credentials) που έχουν δημιουργηθεί αυτόματα από τον Seeder, ή, κάνοντας "Continue with GitHub" με έναν νέο λογαριασμό και δίνοντας στον εαυτό σας Admin δικαιώματα από τη βάση.

Μέσα στο Admin Dashboard παρέχονται οι εξής δυνατότητες:
1. **Διαχείριση Χρονικού Προγραμματισμού (Calendar):** Προβολή και διαχείριση προβολών ταινιών σε ημερολογιακή μορφή.
2. **Διαχείριση Χώρων (Spaces):** CRUD λειτουργίες για τις αίθουσες/θέσεις του κινηματογράφου.
3. **Διαχείριση Χρηστών & Ρόλων:** Αναζήτηση χρηστών, αλλαγή δικαιωμάτων πρόσβασης (Toggle Admin) και διαχείριση συστήματος ρόλων.

### Β. Εγγραφή / Είσοδος Πελατών
Οι απλοί χρήστες μπορούν να εγγραφούν στο σύστημα είτε μέσω φόρμας (παρέχοντας email και κωδικό πρόσβασης) είτε μέσω **GitHub OAuth2 (Single Sign-On)** με ένα μόνο κλικ. Κατά την ταυτοποίηση μέσω GitHub, το σύστημα επικοινωνεί αυτόματα με το API του παρόχου και ανακτά τα απαραίτητα στοιχεία.

### Γ. Καταγραφή Εισόδου και Ταυτοποίηση (Scanner Επισκεπτών)
Το σύστημα υποστηρίζει την ηλεκτρονική ταυτοποίηση πελατών (Mobile-First λειτουργία). Η ροή είναι η εξής:
1. **Καταγραφή Στοιχείων:** Ο επισκέπτης, μέσω της διαδρομής `/entry/register` (η οποία είναι σχεδιασμένη για Mobile οθόνες), καλείται να συμπληρώσει υποχρεωτικά Όνομα, Επώνυμο, Κινητό Τηλέφωνο, Email και Φωτογραφία (η οποία λαμβάνεται απευθείας από την κάμερα της συσκευής).
2. **Σάρωση/Ταυτοποίηση:** Ο υπάλληλος/διαχειριστής του χώρου, μέσω του `/admin/scanner`, συμπληρώνει το καταγεγραμμένο Email ή Κινητό Τηλέφωνο. Το Backend (μέσω του `EntryController`) αναζητά τυχόν ενεργές κρατήσεις. Σε περίπτωση επιτυχίας, εμφανίζεται ένα εντυπωσιακό «Ψηφιακό Εισιτήριο» με τη φωτογραφία του επισκέπτη και λεπτομέρειες για την αίθουσα, την ταινία και ξεκάθαρη απεικόνιση των θέσεων / εισιτηρίων της κράτησής του.

## Υλοποίηση Προδιαγραφών Εργασίας

Παρακάτω αναλύεται ο τρόπος με τον οποίο η παρούσα εφαρμογή καλύπτει τις προδιαγραφές της εργασίας:

### 1. Ημερολόγιο διαθεσιμότητας και κρατήσεων
Υλοποιήθηκε ένα διαδραστικό *Admin Calendar* (`/admin/calendar`) καθώς και ένας αναλυτικός *Κατάλογος* κρατήσεων στο Admin Dashboard. Σε αυτές τις οθόνες εμφανίζονται τα στοιχεία του Πελάτη (Ονοματεπώνυμο, Email), ο Χώρος/Αίθουσα (με κωδικό π.χ. H001), και συνολικά τα ποσοστά **Πληρότητας (Booked)** καθώς και ο βαθμός **Προσέλευσης (% Attended)**, δηλαδή πόσοι επισκέπτες εισήλθαν τελικά στην αίθουσα. Οι διαχειριστές μπορούν να παρακολουθούν αυτό το ιστορικό και το status (Εκκρεμεί / Προσήλθε / Ακυρώθηκε) κάθε κράτησης. Εκπληρώνεται από τον `AdminController` και τη React Component `AdminCalendar.jsx`.

### 2. Φόρμα κράτησης
Η διαδικασία κράτησης επιτρέπει στον εγγεγραμμένο χρήστη να πλοηγηθεί στις Ταινίες, να επιλέξει διαθέσιμη ημερομηνία και ώρα προβολής μέσα από διαδραστικό UI. Κατά την ολοκλήρωση της κράτησης έχει τοποθετηθεί **Checkbox (Toggle)** με τίτλο *"Αποστολή στοιχείων κράτησης με email"*. Εάν επιλεχθεί, ανοίγει νέο πεδίο όπου ο χρήστης εισάγει το email του προσκεκλημένου, εκπληρώνοντας την απαίτηση για προώθηση στοιχείων κτήσης (κωδικός, όνομα, περιγραφή, φωτογραφία). Εκπληρώνεται μέσω της Component `MovieDetails.jsx`.

### 3. Υποσύστημα διαχείρισης εισιτηρίων (Admin Spaces)
Έχει αναπτυχθεί πλήρες CRUD (Create, Read, Update, Delete) περιβάλλον στο `/admin/spaces` για τη διαχείριση Χώρων (Αιθουσών) και Προγραμματισμού Προβολών. Όταν ενας διαχειριστής επιχειρεί να τροποποιήσει ή να διαγράψει έναν χώρο ή μια προβολή, το σύστημα (Laravel Backend `AdminSpaceController`) ελέγχει δυναμικά εάν υπάρχουν ήδη συσχετισμένες κρατήσεις. Αν υπάρχουν, η ενέργεια απορρίπτεται ή εμφανίζεται κατάλληλο προειδοποιητικό μήνυμα για την αποφυγή συγκρούσεων/διαγραφής ενεργών εισιτηρίων.

### 4. Υποσύστημα διαχείρισης χρηστών
Το σύστημα αυθεντικοποίησης συνδυάζει δύο μεθόδους:
- **Native Authentication:** Τοπικό σύστημα με Email, Passwords (Sanctum Tokens).
- **Third-Party SSO (OAuth 2.0):** Ενσωματώθηκε το GitHub API (μέσω `laravel/socialite`) επιτρέποντας στους χρήστες να κάνουν είσοδο και εγγραφή με 1 κλικ (Stateless OAuth flow).
Επιπροσθέτως, η διαχείριση χρηστών και ρόλων ("User", "Admin") υλοποιήθηκε πλήρως στις ρυθμίσεις του Dashboard (`/admin/users`), όπου οι διαχειριστές έχουν την άμεση δυνατότητα να δώσουν ή να αφαιρέσουν ρόλους από τους χρήστες της εφαρμογής, προστατεύοντας με Middleware (Route Roles) τις αντίστοιχες ενέργειες.

### 5. Καταγραφή εισόδου στο χώρο/αίθουσα/ταινία
Υλοποιήθηκε μια ειδικά σχεδιασμένη **Mobile-First** φόρμα στο `/entry/register`. Ο επισκέπτης καταχωρεί υποχρεωτικά *Όνομα, Επώνυμο, Email, Κινητό* και του ζητείται να τραβήξει *Φωτογραφία* (χρήση HTML5 device camera attribute). Το Backend καταχωρεί άμεσα τον επισκέπτη. Στη συνέχεια, μέσω της οθόνης Scanner, ο υπάλληλος αναζητά τον επισκέπτη χρησιμοποιώντας το Κινητό ή το Email του. Αν βρεθεί, το σύστημα εμφανίζει ένα προσωποποιημένο Μήνυμα Καλωσορίσματος (Ψηφιακό Εισιτήριο) με τον Κωδικό Κράτησης, το όνομα, τη φωτογραφία, την αίθουσα, τον τίτλο ταινίας, δίνοντας τεράστια έμφαση, οπτικά, στη **Θέση / Αριθμό Εισιτηρίων** της αίθουσας.

### 6. Αυτοματοποίηση εισόδου και ελέγχου εισιτηρίων στο χώρο
Για την αυτοματοποίηση της ταυτοποίησης, η σελίδα ελέγχου (`/admin/scanner`) ενσωμάτωσε **QR Code Scanner** κατευθείαν στον Browser. Ο πελάτης δείχνει το ψηφιακό του εισιτήριο στο οποίο παράγεται αυτόματα QR Code από το User Profile του. Ο ελεγκτής "σκανάρει" το QR Code χρησιμοποιώντας την κάμερα του κινητού ή του υπολογιστή του (μέσω της front-end βιβλιοθήκης `html5-qrcode`). Το Backend αναγνωρίζει τον πελάτη και την κράτηση ακαριαία, ανοίγει το εισιτήριό του και **"τσεκάρει" αυτόματα** ότι ο πελάτης προσήλθε, τηρώντας έτσι το ψηφιακό ιστορικό παρουσιών της Αίθουσας.
